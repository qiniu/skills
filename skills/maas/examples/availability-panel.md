# 可用性面板输出格式

本文件定义 API 可用性面板的**输出格式模板**，以及面板生成时需要注意的数据处理规则。

接口调用方式见 [references/openapi.json](../references/openapi.json)（路径：`/inapi/v3/stat/log`）。

---

## 输出模板

### 可用性概览

```
## API 可用性面板（{start} ～ {end}）

| 指标              | 数值        |
|-----------------|------------|
| 总请求数           | {total}    |
| 成功请求数         | {success}  |
| 失败请求数         | {failure}  |
| **整体成功率**     | **{rate}%** |
| 客户端错误（4xx）   | {4xx}      |
| 服务端错误（5xx）   | {5xx}      |
```

### 近期失败请求（最多 20 条）

```
## 近期失败请求

| 请求 ID      | 模型     | API Key      | 时间              | 错误码 | 错误信息 |
|------------|---------|-------------|-----------------|------|--------|
| {id}       | {model} | {api_key}   | {start_time}    | {code} | {errors[0]} |
| ...

> 如需查看某次失败请求的完整详情，请提供请求 ID。
```

### 汇总结论

面板末尾附一段简短结论，说明：
- 整体可用性是否正常（≥ 99.5% 为良好，< 99% 需关注）
- 主要失败原因（按错误码分类，哪类占比最大）
- 建议下一步操作（如检查限额、联系支持等）

---

## 数据处理规则

- **成功率计算**：`(total - failure_total) / total × 100`，当 `total = 0` 时展示"暂无数据"
- **失败细分**：`status=client_error` 对应 4xx，`status=server_error` 对应 5xx
- **`errors` 字段**：可能为空数组，此时以 HTTP 状态码推断错误类型
- **`state` vs `status`**：响应字段 `state` 的值是 `success`/`fail`，查询参数 `status` 的枚举是 `failure`/`server_error`/`client_error`，不要混用
- **时间范围**：单次查询不超过 35 天
- **非 chat 类型**：查询图像（`image`）或视频（`video`）日志需传 `server_type` 参数；这两种类型暂不支持按 `status` 过滤，需对 `items[].code` 做客户端聚合
