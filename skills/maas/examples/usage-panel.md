# 用量面板输出格式

本文件定义用量分析面板的**输出格式模板**，以及数据处理时需要注意的规则。

接口调用方式见 [references/openapi.json](../references/openapi.json)（路径：`/inapi/v3/stat/new`、`/inapi/v3/apikeys`）。

---

## 输出模板

### Token 消耗趋势（按模型 + 按天）

```
## Token 用量趋势（{start} ～ {end}）

### {model_name}
| 日期       | {计费项1} ({unit}) | {计费项2} ({unit}) |
|------------|------------------|------------------|
| YYYY-MM-DD | {value}          | {value}          |
| ...
| **合计**   | **{total}**      | **{total}**      |
```

- 计费项名称取响应中的 `items[].name`（如"文本输入 tokens"、"文本输出 tokens"）
- 某天无数据时，该天不出现在 `values` 数组里，展示时补 0
- 单位取 `items[].unit`（如 `k/tokens` = 千 Token）

### API Key 用量概览

```
## API Key 用量概览

| Key 名称  | Key（脱敏）    | 状态    | 累计 Tokens | 日额使用率 | 月额使用率 |
|----------|--------------|--------|------------|---------|---------|
| {name}   | {key}        | {状态}  | {total}    | {daily%} | {monthly%} |
```

- 状态：`enabled=true` → ✅ 启用；`enabled=false` → ❌ 禁用
- 配额 `limit=-1` → 显示"未设限"
- 配额使用率 ≥ 80% 时加粗提示；已禁用的 Key 对应列显示 `—`

### 配额预警（按需）

若某 Key 月额使用率 ≥ 80%，在表格后附一行预警：

```
> ⚠️ **{Key 名称}** 月额已用 {N}%，按当前趋势预计 {日期} 前后触达上限。
```

---

## 数据处理规则

- **计费项名称**：直接使用响应 `items[].name`，不做映射（已是中文名）；若是通过 `bo_usage` key 获取的数据，才需要调用 `/inapi/v3/market/pricingitems` 翻译
- **时间范围**：单次查询建议不超过 35 天
- **粒度选择**：按天 `g=day`，按小时 `g=hour`；默认用 `day`
